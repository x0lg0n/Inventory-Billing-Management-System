## How to Submit a Good Support Request

To help us help you more effectively, please include the following information when submitting support requests:

### For Bug Reports

1. **Description**: Clear description of the issue
2. **Steps to Reproduce**: Detailed steps to reproduce the problem
3. **Expected Behavior**: What you expected to happen
4. **Actual Behavior**: What actually happened
5. **Environment Details**:
   - Operating System
   - Node.js version
   - MongoDB version
   - Browser (for frontend issues)
6. **Screenshots**: If applicable
7. **Logs**: Relevant error logs or console output

### For Feature Requests

1. **Problem Statement**: Clear description of the problem or need
2. **Proposed Solution**: Your idea for how to solve it
3. **Alternatives Considered**: Any alternative approaches you've thought about
4. **Additional Context**: Any other relevant information
